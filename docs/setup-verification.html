üîç Installation Verification

Step 1: Docker is Running

Docker Desktop is installed

Click Start, type "Docker Desktop". You should see it in the search results.

Docker is running

Look for the whale icon üê≥ in the bottom-right corner of your screen (near the clock). It should not be animating.

Docker is installed

Open Terminal and run: docker --version. You should see a version number.

Docker is running

Run: docker ps. If you get an error about "Cannot connect", Docker is not running.

docker-compose is installed

Run: docker-compose --version. You should see a version number.

Step 2: Mkweli Container is Running

Mkweli is started

Double-click the Mkweli AML shortcut and select "Check Status". You should see "running" or "healthy".

Mkweli container is running

Run: docker ps. You should see a container with "mkweli" in the name.

Step 3: Web Interface is Accessible

Login page loads

Open your browser and go to http://localhost:8000. You should see the Mkweli login page.

Can log in successfully

Enter your username and password. Click Login. You should see the dashboard.

Step 4: Features Work Correctly

Quick Check works

Try screening a test name like "John Smith". You should get results (even if no matches).

Sanctions data is loaded

Check the dashboard - it should show the number of entries loaded from each sanctions list (UN, UK, OFAC, EU).

üîß Troubleshooting Guide

If any of the checks above failed, find your problem below and follow the fix.

‚ùå Docker is not installed

What this means:

Docker is a required program that Mkweli needs to run. Without it, Mkweli won't work.

How to fix (Windows):
  1. Go to docker.com/products/docker-desktop
  2. Click "Download for Windows"
  3. Run the downloaded installer
  4. Follow the installation steps
  5. Restart your computer
  6. Open Docker Desktop from the Start menu
  7. Wait for it to finish starting (whale icon stops animating)
How to fix (Ubuntu):
  1. Open Terminal (Ctrl+Alt+T)
  2. Run: sudo apt update && sudo apt install -y docker.io docker-compose
  3. Enter your password when asked
  4. Run: sudo usermod -aG docker $USER
  5. Log out and log back in

‚ùå Docker is not running

What this means:

Docker is installed but not started. Mkweli needs Docker to be running.

How to fix (Windows):
  1. Click the Start button
  2. Type "Docker Desktop"
  3. Click to open it
  4. Wait 1-2 minutes for the whale icon to appear and stop moving
How to fix (Ubuntu):
  1. Open Terminal
  2. Run: sudo systemctl start docker
  3. Check it's running: docker ps

‚ùå Cannot connect to localhost:8000

What this means:

The browser can't reach Mkweli. Either it's not running, or something is blocking the connection.

Things to try:
  1. Wait longer - First startup can take 2-5 minutes. Be patient!
  2. Try alternative address - Instead of localhost:8000, try http://127.0.0.1:8000
  3. Check if Mkweli is running:
    • Windows: Use "Check Status" from the launcher
    • Ubuntu: Run docker ps
  4. Restart Mkweli:
    • Windows: Use "Restart Application" from launcher
    • Ubuntu: cd ~/Mkweli && docker-compose restart
  5. Check firewall - Make sure your firewall isn't blocking port 8000

‚ùå "Permission denied" error (Ubuntu)

What this means:

Your user account doesn't have permission to use Docker.

How to fix:
  1. Open Terminal
  2. Run: sudo usermod -aG docker $USER
  3. Enter your password
  4. Log out completely (click power icon ‚Üí Log Out)
  5. Log back in
  6. Try your command again

‚ùå Login fails with correct password

What this means:

The password might have been changed, or there's a browser issue.

Things to try:
  1. Clear browser cache - Press Ctrl+Shift+Delete
  2. Try a different browser - Try Chrome, Firefox, or Edge
  3. Check caps lock - Make sure caps lock is off
  4. Use default password - If you haven't changed it, try: admin123

‚ùå Application is very slow

What this means:

Your computer may not have enough resources, or Docker needs more memory.

Things to try:
  1. Close other programs - Free up memory by closing unused applications
  2. Check RAM - You need at least 4GB RAM (8GB recommended)
  3. Restart Docker - Sometimes this helps free up resources
  4. Wait for first startup - The first run is always slower while loading data

‚ùå No sanctions data loaded

What this means:

The sanctions data files might be missing or corrupted.

Things to try:
  1. Wait for initialization - First startup loads 160MB+ of data, which takes time
  2. Restart the application - This often resolves data loading issues
  3. Check logs for errors:
    • Windows: Use "View Logs" from launcher
    • Ubuntu: cd ~/Mkweli && docker-compose logs

üö® Emergency: Nothing Works

Complete Reset Procedure

If nothing else works, try a complete reset:

  1. Open the Mkweli launcher and select "Stop Application"
  2. Close Docker Desktop (right-click whale icon ‚Üí Quit)
  3. Restart your computer
  4. Open Docker Desktop and wait for it to fully start
  5. Open Mkweli launcher and select "Start Application"
  6. Wait 5 minutes for everything to initialize
  1. Open Terminal
  2. Run: cd ~/Mkweli && docker-compose down
  3. Run: sudo systemctl restart docker
  4. Restart your computer: sudo reboot
  5. After reboot, open Terminal
  6. Run: cd ~/Mkweli && docker-compose up -d
  7. Wait 5 minutes for everything to initialize
